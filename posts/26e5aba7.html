Git和GitHup多人协同项目教程笔记

Git和GitHup多人协同项目教程笔记

本笔记基于和朋友开发一个微信小程序，多人协同，规范开发步骤。

一、初始设置

  1. 项目拉取到本地
  2. 导入微信开发者工具
  3. 设置认证密码
  4. 设置本地分支

（一）项目拉取到本地

  • 新建项目文件夹

    在存放项目代码的文件夹中创建一个SignLanguageLaw-mp文件夹。

  • 从GitHub拉取项目代码

    需要确保git设置好了远程连接，推荐使用SSH协议的方式连接认证。

方法一：使用命令行拉取

​ 打开创建的项目文件夹，在SignLanguageLaw-mp文件夹里面【右键】打开【Git Bash Here】命令行，输入git clone命令：

git clone 仓库地址

方法二：使用GitHubDesktop软件拉取代码

​ 打开GitHubDesktop软件

  • 检查项目文件夹：SignLanguageLaw-mp

（二）导入微信开发者工具

打开【微信开发者工具】，微信扫码登录（微信已经添加开发者）。

  • 【导入】，选择项目文件目录，SignLanguageLaw-mp文件夹；注意：【后端服务】选择【不使用云服务】。

  • 导入成功之后就会直接进入编辑器。

（三）设置认证密码

  • 微信开发者工具： 将其修改为自己的GitHub账户和密码
  • 顺便检查一下远程是否正确

（四）设置本地分支

注意：分支我已经在远程创建了三个，一人一个。

  • dev_lx

  • dev_lh

  • dev_lyy

方法一：命令设置

  • 项目目录下打开，创建dev_lx分支。

    命令：git checkout -b dev_lx

    ​ git push origin dev_lx:dev_lx

  • 一些命令

    拉取 ：git pull

    本地创建分支：git checkout -b 分支名

    查看本地分支：git branch -a

    切换本地分支：git checkout 分支名

方法二：微信开发者工具直接创建

  • 如下图，在【本地】右键【创建分支】

  • 提交分支，打开【GitHubdesktop】上面发布该分支到远程

  • 切换分支，在新创建的分支上面右键，第一个【检出】
  • 切换检查，分支后面会有有一个HEAD，表示当前在使用该分支；
  • 或者打开项目文件夹目录，打开【Git Bash Here】，地址后面会有一个括号，标明当前使用分支。

  • 至此本地基本配置完毕。

二、开发项目步骤

  • 抓取
  • 开发
  • 上传

（一）抓取

  • 每天开始做项目前，先抓取main主分支中的最新代码

方法一：命令抓取。（不写了）

方法二：GitHubdesktop软件

方法三：微信开发者工具直接抓取

（二）开发

  • 保证在自己的本地分支下进行开发！！！

（三）上传

  • 每天结束代码工作对代码进行上传到GitHub数据仓库

  • 上传

    方法一：命令上传 （不写了）

    方法二：GitHubDesktop上传

​ 方法三：微信开发者工具直接上传

（四）开发日志

  • 项目上传之前编写一下自己的开发日志，方便后续项目维护和项目总结。
  • 项目目录中的workLogs文件夹为个人开发日志文件夹，每个人有一个以自己分支命名的.md文件夹。
  • 日志文件编辑按照文件中模板编写，简要说明就行，简单写一下加入了什么页面以及对应的文件夹。
  • 编写完日志文件一起上传提交代码。

注：

  • 微信开发者工具那个抓取上传如果报错认证信息有问题，就是网络认证那有问题。

  • 抓取和上传直接使用GitHubDesktop比较好，不使用微信开发者工具里面的

  • 要写代码的话，一定要去抓取同步一下main主分支，如果别人上传了合并了新的代码才能保证自己本地的开发代码是最新的。

  • 关于每天打代码结束上传代码，建议如果代码有一个报错还没有解决的话就不要上传先，如果上传合并到主分支了，别人在同步开发的时候就会出现对方不了解的报错问题。
